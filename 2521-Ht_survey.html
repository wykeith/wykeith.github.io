Height Survey Analysis

Posted on Sun 09 September 2018 in Blog

Datascience exercise #1¶

Attended a basic R tutorial for my MUSPP class, everyone was tasked to estimate the heights of each other and record them into a table. This formed the dataset which we subsequently analysed using R. I'm repeating this exercise in Python as it is easier for me for analysis. Since I cant use python as the class assignments are based on R, I will try to do the python recoding for all my future assignments on this blog.

Observations¶

  • Generally most of us under-estimate
  • With the exception of MaKaKu who tend to mostly over-estimate
  • Yours truly understimates everyone's height, and since I'm consistently inaccurate, one could add 10cm to all future predictions from myself to achieve an estimate that is closer to reality. Maybe I have got parallex error in my vision.
  • Plotting the all errors for each estimates individually, we also find that for the shortest (DeKo), most of the estimators over-estimate, ie more blue bars. And for the tallest (LiSwKi) most of the estimators under-estimate, ie more orange bars. See chart below

Takeaway if estimators were IOT sensors, the above chart can be used to visualise under-estimation/over-estimations during calibration.

Key Insight¶

From the limited observations everyone is generally good at estimating heights between 165-170. There is a strong correlation r-value = -0.92 between errors and actual heights. Suggesting that that there is over-estimating for shorter people and underestimating for taller people.
